Why This Doesn't Work

This conversation took 6 minutes when it could have taken 30 seconds. Developer B had to ask multiple clarifying questions before understanding what was needed.

When you ask vague questions about code, you're forcing the other developer to:

  • Stop their current work to figure out what you need
  • Ask multiple clarifying questions
  • Potentially search through their projects to find the right script

Context is Key

Always provide enough context in your initial message so the other developer can understand exactly what you need.

Other vague requests to avoid:

  • "Can you share that script?"
  • "How did you fix the bug yesterday?"
  • "Send me the GUI code"
  • "I need help with my script"

Do This Instead

A

Hi! I'm working on my obby game and having trouble with teleportation. Could you share the teleport script from your Rainbow Obby that handles checkpoint respawning? I'm specifically interested in how you manage the player's last checkpoint.

local function onPlayerAdded(player)
    local checkpoint = Checkpoints:FindFirstChild("Start")
    player.CharacterAdded:Connect(function(character)
        -- Etc Code
    end)

Players.PlayerAdded:Connect(onPlayerAdded)

Why This Works Better

By providing context upfront, Developer A respected Developer B's time and made it easy to provide the right code immediately.

Developer B could immediately understand:

  • What project the code was needed for (obby game)
  • Which specific script was needed (teleport/checkpoint script)
  • From which game (Rainbow Obby)
  • The specific functionality (handling player's last checkpoint)

Good Code Requests Include:

  • The specific script or function you need
  • Which game/project it's from
  • What you're trying to accomplish
  • Any error messages or issues you're facing

More examples of good requests:

  • "Can you share the tool script from your Adventure game that handles the double jump ability?"
  • "I'm getting 'attempt to index nil with Player' error in my leaderboard script. How did you handle player data loading in your RPG?"
  • "The GUI for my shop isn't scaling properly on mobile. Could I see how you set up your responsive UI in your Tycoon game?"

When you provide context upfront, everyone wins! ðŸŽ‰

Try using `TeleportService:TeleportToPlaceInstance()` instead for better reliability:

lua
local TeleportService = game:GetService("TeleportService")

function teleportPlayer(player, placeId, spawnName)
    local teleportOptions = Instance.new("TeleportOptions")
    teleportOptions.ShouldReserveServer = false
    teleportOptions.SpawnLocationName = spawnName
    
    TeleportService:TeleportToPlaceInstance(placeId, game.JobId, player, teleportOptions)
end

Discord Formatting Best Practices

  • Use ```lua before your code and ``` after for proper syntax highlighting
  • Use `inline code` for short code snippets or function names
  • Include error messages and relevant context
  • Keep code blocks focused and relevant to your question

Why Proper Formatting Matters

When you format your Lua code correctly in Discord, you make it much easier for others to help you:

  • Syntax Highlighting: Properly formatted code is easier to read and understand
  • Clear Structure: Code blocks maintain proper indentation and formatting
  • Mobile Friendly: Formatted code is more readable on all devices
  • Professional Appearance: Shows you've put effort into your request
